        - 중요! 해당 조건을 만족하는 부분을 캡쳐해 근거로 첨부
    
- [O]  **2. 전체 코드에서 가장 핵심적이거나 가장 복잡하고 이해하기 어려운 부분에 작성된 
주석 또는 doc string을 보고 해당 코드가 잘 이해되었나요?**
    데이터 전처리(리사이즈·레이블링)와 CNN 모델 구조 정의 부분이 핵심인데, 해당 블록에 주석이 잘 달려 있어 코드의 역할과 동작 방식을 쉽게 이해할 수 있었다. 
    특히 입력 데이터 흐름과 모델 계층 구성 원리가 명확하게 설명되어 있어 전체 코드 이해에 도움이 되었다.
        
- [O]  **3. 에러가 난 부분을 디버깅하여 문제를 해결한 기록을 남겼거나
새로운 시도 또는 추가 실험을 수행해봤나요?**
    코드 실행 중 발생한 “No valid data after preprocessing” 문제를 원인(데이터 경로·전처리 오류)과 함께 기록하고 수정해 해결했다.
    또한 왼손·오른손 데이터 분리 문제, 해상도 변경, 데이터 증강 적용 등 추가 실험을 시도하며 성능 변화를 비교한 기록이 포함돼 있어 개선 과정을 명확하게 보여준다.
          
- [O]  **4. 회고를 잘 작성했나요?**
    프로젝트 전반에서 데이터 한계, 해상도 문제, 손 모양 차이로 인한 일반화 실패 등 배운 점과 아쉬운 점을 명확하게 정리해 회고가 충실하게 작성되어 있다.
    또한 전체 코드 흐름을 단계별로 설명하며 이해를 돕고 있어 실행 플로우를 명확히 파악할 수 있었다.
            
- [O]  **5. 코드가 간결하고 효율적인가요?**
    이미지 리사이즈·데이터 로딩·정규화 등을 함수로 분리해 중복을 줄인 점이 효율적이고, 불필요한 반복문이나 하드코딩을 최소화하여 구조가 명확하고 재사용 가능성이 높다. 
    특히 여러 번 시도하며 수정한 내용들을 최종 코드에 정리해 흐름이 한눈에 들어오도록 구성한 점이 좋았다.
